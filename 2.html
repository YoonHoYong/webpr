css

In the past, it was common to include design elements in HTML. In other words, putting all kinds of layout and design information into HTML has turned it into a document for design, not a structured document, which is the original purpose of HTML. The HTML source code alone made it difficult for even experts to know which document this document was, for example, the Table tag to draw up the table was used to construct the layout. Therefore, the W3C announced the CSS with the aim of "Let's separate design elements completely from HTML to create structured HTML!" Accordingly, HTML recommends that all elements related to design be avoided, such as bgcolors. After the CSS announcement, new tags such as div, span, and section to help structure HTML documents were introduced, and style tags such as strike and font were switched to non-recommended tags and deleted altogether in HTML5. The table has been newly introduced to help structure the table, recommending that it be used only for expressing data in tabular form. Structured HTML is the use of HTML tags for their intended use. For example, the h1 tag, which stands for the top title, is written at the level of the title of the document, and the text paragraph is divided by the

tag. This makes HTML much easier to recognize because most design elements are excluded. Here, 'easy to know' includes not only people (programmers), but also machines (web browsers). HTML, which is easy for machines to understand, is easy to change output format into various formats (which are applied by preparing and replacing CSS files separately for mobile devices and PCs), speed of decoding (packing) of documents is faster, and it is easier to implement dynamic functions. And in most cases, structured HTML is relatively smaller in document size than unstructured HTML in the past.
